# Contributing to MyDigialLibrary

Thank you for considering contributing! Here are some guidelines and helpful tips to make the process smooth.

---

## Ways To Contribute

- Reporting bugs with the bug template.
- Submitting feature requests or enhancements.
- Contributing code (bug fixes, features, refactoring).
- Improving documentation.

---

## Getting Started

1. Fork and clone the repository.
2. Set up your dev environment (see `/docs/`).
3. Create a feature or fix branch.
4. Write and test your changes.
5. Submit a pull request using the PR template.

---

## Development Requirements

- Code should be readable and maintainable.
- Write tests for new features or bug fixes.
- Use descriptive commit messages.
- Keep pull requests focused and atomic.

---

## Code Review

- At least one approval required before merging.
- Address requested changes promptly.
- All CI checks must pass.

---

## Code of Conduct

Be respectful, collaborative, and open to feedback.

---

## Questions?

Open an issue or join the project discussions!
